# 🤖 PROMPTS_USED.md

## 📌 Overview

This document outlines how AI tools were used during the development of the **SnackSprint – School Canteen Management System** project.

The purpose of using AI was to:
- Improve development speed
- Enhance UI/UX quality
- Follow best practices in frontend development
- Debug and optimize code efficiently

---

## 🧠 AI Tools Used

- **ChatGPT (OpenAI)**  
  Used for code generation, debugging, UI improvement suggestions, and architectural guidance.

---

## 💬 Types of Prompts Used

### 1️⃣ Project Setup & Architecture
- Asked for:
  - React project structure
  - Folder organization
  - Component breakdown
- Helped in designing a scalable and clean architecture

---

### 2️⃣ Mock API & Backend Simulation
- Prompts related to:
  - Creating `db.json`
  - Setting up `json-server`
  - Designing REST API endpoints
- Result:
  - Fully functional mock backend for development

---

### 3️⃣ State Management (Zustand)
- Guidance for:
  - Creating global store
  - Managing snacks, students, and orders
  - Handling async API calls
- Result:
  - Efficient and centralized state management

---

### 4️⃣ UI/UX Design Improvements
- Asked for:
  - Modern cafeteria-style UI
  - Responsive layouts
  - Card-based design
  - Modal improvements
- Result:
  - Clean and user-friendly interface

---

### 5️⃣ Error Handling & Edge Cases
- Prompts included:
  - Fixing undefined errors
  - Handling API failures
  - Adding loading states
- Result:
  - Stable and production-like UI behavior

---

### 6️⃣ Feature Enhancements
- Helped implement:
  - Student search functionality
  - Order system with validation
  - Auto referral code generation
  - Incremental ID logic
- Result:
  - Feature-rich application

---

### 7️⃣ Debugging Support
- Issues resolved:
  - React hook errors
  - Zustand misuse
  - API integration bugs
  - State not updating
- Result:
  - Smooth working application

---

## ⚙️ How AI Was Used

- AI suggestions were **reviewed, modified, and integrated manually**
- Code was **tested locally before implementation**
- Final implementation reflects **developer understanding and customization**

---

## 📌 Important Note

This project was **not blindly generated using AI**.  
AI was used as a **development assistant**, while:
- Logic building
- Integration
- Debugging decisions
were actively handled by the developer.
